# 腾讯云轻量服务器部署指南

## 1. 目录结构

```text
/srv/apps/zjuaaa-site           # 代码
/srv/data/zjuaaa-site/dev.db    # 数据库
/srv/data/zjuaaa-site/uploads   # 上传图片
/srv/data/zjuaaa-site/.env      # 环境变量
/srv/backups/zjuaaa-site        # 备份
```

---

## 2. 环境要求

- 系统：`Ubuntu 22.04 LTS`
- Node：`22.x`
- 进程管理：`PM2`
- 反向代理：`Nginx`

---

## 3. 首次部署

### 3.1 安装基础软件

```bash
apt update && apt upgrade -y
apt install -y git curl unzip nginx sqlite3
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
npm install -g pm2
```

### 3.2 创建目录

```bash
mkdir -p /srv/apps
mkdir -p /srv/data/zjuaaa-site/uploads
mkdir -p /srv/backups/zjuaaa-site
```

### 3.3 拉取源码

```bash
cd /srv/apps
git clone 你的仓库地址 zjuaaa-site
cd /srv/apps/zjuaaa-site
```

### 3.4 创建环境变量

生成随机串：

```bash
openssl rand -base64 32
```

创建 `.env` 文件（用你熟悉的编辑器，或直接执行以下命令）：

```bash
cat > /srv/data/zjuaaa-site/.env << 'EOF'
DATABASE_URL="file:/srv/data/zjuaaa-site/dev.db"
AUTH_SECRET="替换为你的随机字符串"
AUTH_URL="https://zjuaaa.cn"
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="替换为你的后台密码"
ADMIN_DISPLAY_NAME="ZJUAAA Admin"
NODE_ENV="production"
UPLOAD_DIR="/srv/data/zjuaaa-site/uploads"
EOF
```

### 3.5 链接 .env 到代码目录

```bash
rm -f /srv/apps/zjuaaa-site/.env
ln -s /srv/data/zjuaaa-site/.env /srv/apps/zjuaaa-site/.env
```

### 3.6 安装依赖并初始化

```bash
cd /srv/apps/zjuaaa-site
npm install
npx prisma generate
npx prisma db push --skip-generate
npm run db:seed-admin
npm run db:seed-categories
```

### 3.7 构建并启动

```bash
npm run build
pm2 start npm --name zjuaaa-site -- start
pm2 save
pm2 startup
```

## 4. Nginx 配置

### 4.1 创建配置文件

```bash
cat > /etc/nginx/sites-available/zjuaaa-site << 'EOF'
server {
    listen 80;
    server_name zjuaaa.cn 49.235.184.100;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name zjuaaa.cn 49.235.184.100;

    ssl_certificate /etc/letsencrypt/live/zjuaaa.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/zjuaaa.cn/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;

    client_max_body_size 64M;

    location /uploads/ {
        alias /srv/data/zjuaaa-site/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        autoindex off;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
```

### 4.2 启用配置

```bash
rm -f /etc/nginx/sites-enabled/default
ln -sf /etc/nginx/sites-available/zjuaaa-site /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

### 4.3 申请 HTTPS 证书（如果还没有）

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d zjuaaa.cn
```

证书会自动续期。申请完成后，重新执行步骤 4.1 和 4.2 恢复自定义配置。

## 7. 数据库迁移

如果 `schema.prisma` 有变化（新增表或字段），更新代码后需要：

```bash
cd /srv/apps/zjuaaa-site

# 同步数据库结构（自动创建表、添加字段）
npx prisma db push --skip-generate

# 如果新增了非空字段且数据库已有数据，需要先设置默认值
# 例如：给旧文章指定默认栏目
sqlite3 /srv/data/zjuaaa-site/dev.db "UPDATE ManualChapter SET categoryId = (SELECT id FROM ManualCategory WHERE slug='observation') WHERE categoryId IS NULL;"

# 初始化新功能的默认数据（如果有的话）
npm run db:seed-categories
```

**说明：** `npx prisma db push` 会根据 `schema.prisma` 自动同步数据库结构，不需要手操 SQL 创建表或添加字段。

## 11. 故障排查

### 查看错误日志

```bash
pm2 logs zjuaaa-site --err --lines 50
```

### 常见问题

| 问题 | 排查方式 |
|------|---------|
| 页面 500 | 查看 PM2 日志，通常是数据库字段缺失 |
| 登录失败 | 检查 `AUTH_URL` 是否正确，运行 `npm run db:seed-admin` |
| 图片上传失败 | 检查 `UPLOAD_DIR` 目录权限，检查 Nginx `client_max_body_size` |
| 数据库字段缺失 | 运行 `npx prisma db push --skip-generate`，或手动执行 SQL |
| Nginx 502 | 检查 PM2 是否在运行，检查端口 3000 |

### 手动执行 SQL

```bash
sqlite3 /srv/data/zjuaaa-site/dev.db

# 查看表结构
.schema ManualChapter

# 查看数据
SELECT * FROM ManualCategory;

# 退出
.quit
```
