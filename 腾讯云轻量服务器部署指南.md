# ZJUAAA 网站部署指南（腾讯云轻量应用服务器 / Ubuntu 22.04）

这份文档默认你已经买好了：

- 腾讯云轻量应用服务器
- 镜像选择：`Ubuntu 22.04 LTS`

本文适用于当前这个项目：

- 前端：`Next.js`
- 后端接口：`Next.js Server Actions / App Router`
- 数据库：`SQLite`
- 图片上传：本地 `public/uploads`
- 进程管理：`PM2`
- 反向代理：`Nginx`

---

## 1. 这台服务器够不够

对于你们现在这个社团官网，这台配置可以上线：

- `2核 2G`
- `40GB SSD`
- `200GB/月流量`
- `3M 带宽`

适合当前阶段：

- 官网展示
- 后台登录与内容管理
- 图片数量中等
- 日常访问量不算特别大

注意：

- 图片越来越多以后，最先紧张的通常是 `磁盘空间` 和 `月流量`
- 如果以后摄影大图很多，建议后续把图片迁移到对象存储
- `SQLite` 适合你们现在这版，但一定要做备份

## 2. 服务器准备

先用腾讯云控制台获取：

- 公网 IP
- root 密码

然后在本地用终端连接服务器：

```bash
ssh root@你的服务器公网IP
```

第一次连接时输入 `yes`，再输入密码。

## 4. 上传项目到服务器

你们现在有两种常见方式。

### 方式 A：上传整个项目文件夹

如果你们项目还没推到 GitHub，这是最直接的方式。

在本地把项目上传到服务器，例如用：


## 6. 配置环境变量

在项目根目录创建 `.env`：

```bash
nano .env
```

写入下面内容：

```env
DATABASE_URL="file:./prisma/dev.db"
AUTH_SECRET="这里替换成一长串随机字符串"
AUTH_URL="https://你的域名"
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="你要设置的后台初始密码"
ADMIN_DISPLAY_NAME="ZJUAAA Admin"
NODE_ENV="production"
```

说明：

- `AUTH_SECRET` 要自己换成随机字符串，越长越好
- 如果你还没绑定域名，临时也可以先写：
  - `AUTH_URL="http://服务器公网IP"`
- `ADMIN_PASSWORD` 建议不要再用默认密码

随机生成密钥可以用：

```bash
openssl rand -base64 32
```

## 7. 初始化数据库

当前项目使用 SQLite，本地数据库文件会放在项目里。

先同步数据库结构：

```bash
npx prisma db push --skip-generate
```

生成 Prisma Client：

```bash
npx prisma generate
```

创建初始管理员：

```bash
npm run db:seed-admin
```

如果你改了 `.env` 里的管理员用户名和密码，这里会按 `.env` 创建。

## 10. 配置 Nginx

创建 Nginx 配置文件：

```bash
nano /etc/nginx/sites-available/zjuaaa-site
```

写入：

```nginx
server {
    listen 80;
    server_name 你的域名 你的服务器公网IP;

    client_max_body_size 25M;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/zjuaaa-site /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

## 12. 绑定域名

到你的域名服务商后台，把域名解析到腾讯云服务器公网 IP：

- `A 记录`
- 主机记录：`@`
- 值：你的公网 IP

如果还想用 `www`：

- `A 记录`
- 主机记录：`www`
- 值：你的公网 IP

等解析生效后，浏览器打开：

```text
http://你的域名
```

---

## 13. 配置 HTTPS

推荐用 `certbot`。

安装：

```bash
apt install -y certbot python3-certbot-nginx
```

申请证书：

```bash
certbot --nginx -d 你的域名 -d www.你的域名
```

成功后它会自动帮你改 Nginx。

完成后记得把 `.env` 里的：

```env
AUTH_URL="https://你的域名"
```

改成正式 HTTPS 域名。

改完后重启服务：

```bash
pm2 restart zjuaaa-site
```

## 15. 这个项目当前重要数据存在哪里

### 数据库

```text
/var/www/zjuaaa-site/prisma/dev.db
```

### 上传图片

```text
/var/www/zjuaaa-site/public/uploads
```

也就是说：

- 文章、活动、摄影、手册、管理员账号都在 `dev.db`
- 你们后台上传的图片在 `public/uploads`

这两处一定要备份。

---

## 16. 备份建议

最少备份这两个位置：

```bash
cp /var/www/zjuaaa-site/prisma/dev.db /var/backups/zjuaaa-dev-$(date +%F).db
cp -r /var/www/zjuaaa-site/public/uploads /var/backups/zjuaaa-uploads-$(date +%F)
```

建议：

- 每次大更新前备份一次
- 每周备份一次数据库
- 如果摄影图片越来越多，考虑后续迁移到腾讯云 COS

## 18. 当前项目部署上的特别提醒

这个项目当前有几个部署层面的特点：

### 1. 使用 SQLite

优点：

- 简单
- 上线快
- 适合社团官网

缺点：

- 不适合很高并发
- 不适合多机部署
- 迁移服务器时要一起带走数据库文件

### 2. 图片存在本地

优点：

- 现在最省事
- 后台上传后立刻可用

缺点：

- 会占服务器磁盘
- 换服务器时要迁移上传目录

### 3. 后台账号密码依赖 `.env`

第一次初始化管理员时，会读取：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_DISPLAY_NAME`

所以第一次上线前要先想好后台初始密码。

## 20. 如果部署时卡住，最常见排查点

### 构建失败

先看：

```bash
npm run build
```

### 后台打不开

检查：

- `.env` 里的 `AUTH_SECRET`
- `.env` 里的 `AUTH_URL`
- 是否执行过：

```bash
npx prisma generate
npm run db:seed-admin
```

### 图片上传失败

检查：

- `public/uploads` 是否存在
- 目录权限是否正常
- Nginx 的 `client_max_body_size` 是否足够

### 页面 502 / 504

检查：

```bash
pm2 logs zjuaaa-site
systemctl status nginx
```
