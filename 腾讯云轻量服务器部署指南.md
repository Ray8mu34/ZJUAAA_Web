# 腾讯云轻量服务器部署指南

## 1. 推荐部署结构

```text
/srv/apps/
├── zjuaaa-site

/srv/data/
├── zjuaaa-site/
│   ├── dev.db
│   ├── uploads/
│   └── .env

/srv/backups/
├── zjuaaa-site/
```

说明：

- `/srv/apps/zjuaaa-site` 放代码
- `/srv/data/zjuaaa-site` 放数据库、上传图片和环境变量
- `/srv/backups/zjuaaa-site` 放备份

## 2. 推荐环境

- 系统：`Ubuntu 22.04 LTS`
- Node：`22.x`
- 进程管理：`PM2`
- 反向代理：`Nginx`

## 3. 首次部署

### 安装基础软件

```bash
apt update && apt upgrade -y
apt install -y git curl unzip nginx
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
npm install -g pm2
```

### 创建目录

```bash
mkdir -p /srv/apps
mkdir -p /srv/data/zjuaaa-site/uploads
mkdir -p /srv/backups/zjuaaa-site
```

### 拉取源码

```bash
cd /srv/apps
git clone 你的仓库地址 zjuaaa-site
cd /srv/apps/zjuaaa-site
```

### 创建环境变量

```bash
nano /srv/data/zjuaaa-site/.env
```

参考内容：

```env
DATABASE_URL="file:/srv/data/zjuaaa-site/dev.db"
AUTH_SECRET="请自行生成一个足够长的随机字符串"
AUTH_URL="https://你的域名"
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="你的后台初始密码"
ADMIN_DISPLAY_NAME="ZJUAAA Admin"
NODE_ENV="production"
UPLOAD_DIR="/srv/data/zjuaaa-site/uploads"
```

生成随机串：

```bash
openssl rand -base64 32
```

### 链接 `.env`

```bash
rm -f /srv/apps/zjuaaa-site/.env
ln -s /srv/data/zjuaaa-site/.env /srv/apps/zjuaaa-site/.env
```

### 安装依赖并初始化

```bash
cd /srv/apps/zjuaaa-site
npm install
npx prisma db push --skip-generate
npx prisma generate
npm run db:seed-admin
```

### 构建并启动

```bash
npm run build
pm2 start npm --name zjuaaa-site -- start
pm2 save
pm2 startup
```

查看状态：

```bash
pm2 status
pm2 logs zjuaaa-site --lines 50
```

## 4. Nginx 反向代理

编辑配置：

```bash
nano /etc/nginx/sites-available/zjuaaa-site
```

参考内容：

```nginx
server {
    listen 80;
    server_name 你的域名 www.你的域名;

    client_max_body_size 64M;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/zjuaaa-site /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

## 5. HTTPS

建议用 `certbot`：

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名 -d www.你的域名
```

## 6. 图片上传说明

### 上传路径

运行时图片不再写入 `public/uploads`，而是写到：

```text
/srv/data/zjuaaa-site/uploads
```

访问地址仍然保持：

```text
/uploads/文件名
```

### 当前大小限制

- Next Server Actions 请求体限制：`64MB`
- 单张图片上传限制：`50MB`

如果你上传大图，记得同时保证：

- `.env` 中 `UPLOAD_DIR` 正确
- `Nginx` 里 `client_max_body_size 64M`

## 7. 媒体库分类的自动行为

现在有两个自动联动：

### 内建

如果在媒体库上传时分类选：

- `内建`

则图片会自动加入：

- 关于我们照片墙

### 天文摄影

如果在媒体库上传时分类选：

- `天文摄影`

则系统会自动：

- 创建一条摄影作品记录
- 默认作者：`天小协`
- 标题自动生成
- 状态默认直接发布

这很适合批量上传摄影作品。

## 8. 日常更新代码

以后网站更新时，一般只需要：

```bash
cd /srv/apps/zjuaaa-site
git pull
npm install
npx prisma generate
npm run build
pm2 restart zjuaaa-site
pm2 logs zjuaaa-site --lines 30
```

如果 `schema.prisma` 有变化，再补：

```bash
npx prisma db push --skip-generate
```

## 9. 备份

### 备份数据库

```bash
cp /srv/data/zjuaaa-site/dev.db /srv/backups/zjuaaa-site/dev-$(date +%F-%H%M%S).db
```

### 备份上传图片

```bash
tar -czf /srv/backups/zjuaaa-site/uploads-$(date +%F-%H%M%S).tar.gz /srv/data/zjuaaa-site/uploads
```

## 10. 上线后建议检查

- 首页是否能打开
- 后台是否能登录
- 媒体库是否能上传图片
- `/uploads/文件名` 是否能访问
- 天文手册公式是否正常渲染
- PM2 是否正常运行
- HTTPS 是否已生效

如果这些都正常，基本就可以稳定上线使用了。
