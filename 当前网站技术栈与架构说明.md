# 当前网站技术栈与架构说明

本文用于后续维护、项目交接与二次开发。

---

## 第一部分：网站技术栈

### 1. 总体架构

本项目采用 `Next.js App Router` 一体化架构，同一个仓库内同时包含：

- 前台页面
- 后台管理页面
- 服务端接口
- 数据库访问逻辑
- 登录认证逻辑

可以理解为：一个以 `Next.js` 为核心的全栈网站项目，适合内容型网站、学生社团官网、轻量后台管理场景。

---

### 2. 前端技术栈

- `Next.js 15`
- `React 19`
- `TypeScript`
- 原生 `CSS`
- `next/image` 用于图片显示

前端特点：

- 前台与后台都写在 `src/app` 下
- 使用 React 组件拆分页面结构
- 样式集中在 `src/app/globals.css`

---

### 3. 后端技术栈

- `Next.js Server Actions`
- `Next.js Route Handlers`
- `NextAuth v5 beta`

后端职责主要包括：

- 管理后台表单提交
- 登录鉴权
- 上传文件访问
- 图片缩略图 / 水印处理

---

### 4. 数据库与 ORM

- `Prisma`
- `SQLite`

当前开发环境默认数据库：`prisma/dev.db`

当前数据模型主要包括：

- `AdminUser`：管理员账号
- `SiteSetting`：站点设置
- `MediaAsset`：媒体库
- `KnowledgePost`：科普文章
- `ManualCategory`：手册栏目
- `ManualChapter`：手册文章
- `ActivityNotice`：活动
- `AstroPhoto`：天文摄影作品

---

### 5. 认证与权限

- 使用 `NextAuth`
- 登录方式为账号密码登录（`Credentials Provider`）
- 后台页面访问控制由两层实现：
  1. `middleware.ts` 做路由级拦截
  2. `src/lib/admin-session.ts` 做服务端兜底校验

---

### 6. 图片与文件处理

网站图片系统分为三层：

#### 6.1 原始上传文件

- 对外路径：`/uploads/xxx`
- 实际目录由环境变量 `UPLOAD_DIR` 控制

#### 6.2 图片变体访问

- `/media/...` 用于图片代理访问
- 支持 `thumb`（缩略图）和 `original`（原图）

#### 6.3 水印逻辑

原图查看支持水印处理，内容为：`浙江大学学生天文爱好者协会 ZJUAAA`

---

### 7. Markdown / 手册渲染

天文手册页面使用：

- `react-markdown`
- `remark-gfm`
- `remark-math`
- `rehype-katex`
- `katex`

功能：支持 Markdown、公式、站内图片插入。

---

### 8. 部署环境

推荐线上环境：

- `Ubuntu 22.04`
- `Node.js 22.x`
- `PM2`
- `Nginx`

## 第三部分：业务模块说明

### 1. 首页

负责对外展示站点气质与核心内容入口。

### 2. 科普

以卡片形式展示文章摘要，支持跳转外部原文或站内详情。

### 3. 活动

展示社团活动信息与外链。

### 4. 天文摄影

展示摄影作品列表、详情弹窗、详情页。

### 5. 天文手册

分栏目管理的站内内容系统，支持 Markdown 与公式。

**结构：**
- 栏目（ManualCategory）：如"观测手册"、"天文学入门"等
- 文章（ManualChapter）：属于某个栏目，支持排序、发布状态

**URL 结构：**
- 栏目总览：`/manual`
- 栏目文章列表：`/manual/[category]`
- 文章详情：`/manual/[category]/[chapter]`

**功能：**
- 后台栏目管理：增删改栏目、设置封面图、控制是否对外可见
- 后台文章管理：按栏目筛选、拖拽上传图片、批量导入
- 批量导入：上传 ZIP 文件一次性导入整个栏目的文章和图片

### 6. 关于我们

展示社团介绍、照片墙、历届成员。

### 7. 后台管理

负责维护以上所有模块内容。

## 第四部分：数据与文件流转说明

### 1. 图片上传流程

1. 后台媒体库上传图片（或编辑器内拖拽上传）
2. 图片写入上传目录
3. 数据库存入 `MediaAsset.filePath`
4. 前台或后台预览时优先走缩略图
5. 查看原图时走原图变体

### 2. 管理员登录流程

1. 访问 `/admin`
2. `middleware.ts` 先判断是否已登录
3. 页面服务端再通过 `admin-session.ts` 做二次校验
4. 登录成功后进入后台

### 3. 页面内容维护方式

| 内容 | 数据模型 |
|------|---------|
| 站点基础信息 | `SiteSetting` |
| 科普文章 | `KnowledgePost` |
| 活动 | `ActivityNotice` |
| 摄影 | `AstroPhoto` |
| 手册栏目 | `ManualCategory` |
| 手册文章 | `ManualChapter` |
| 媒体库 | `MediaAsset` |

## 第五部分：维护注意事项

### 1. 不要随意删除这几个文件

- `src/lib/auth.ts`
- `src/lib/admin-session.ts`
- `middleware.ts`
- `src/app/media/[...path]/route.ts`
- `prisma/schema.prisma`

### 2. 图片系统说明

- `/uploads/...` 是原始文件
- `/media/...` 是站内图片变体入口

### 3. 线上环境变量必须正确

关键变量：

- `DATABASE_URL`
- `UPLOAD_DIR`
- `AUTH_SECRET`
- `NEXTAUTH_SECRET`（或 `AUTH_SECRET`）
- `NEXTAUTH_URL`（或 `AUTH_URL`）

### 4. 登录问题排查

- Nginx 是否正确转发 `Host` / `X-Forwarded-*`
- `NEXTAUTH_URL` / `AUTH_URL` 是否正确
- 数据库里管理员是否存在
- 是否重新执行过 `seed-admin.mjs`

### 5. 数据库结构修改

修改 `schema.prisma` 后：

1. 本地运行 `npx prisma db push`
2. 服务器同步执行相同操作
3. 如果新增了非空字段，需要先为现有数据设置默认值
