# 线上迁移执行清单

这份文档只保留“直接执行的命令”和少量说明。  
适用于把线上项目整理成下面这种结构：

```text
/srv/apps/zjuaaa-site
/srv/data/zjuaaa-site
/srv/backups/zjuaaa-site
```

---

## 1. 第一次迁移：创建目录

```bash
mkdir -p /srv/apps
mkdir -p /srv/data/zjuaaa-site/uploads
mkdir -p /srv/backups/zjuaaa-site
```

---

## 2. 第一次迁移：拉代码

如果服务器上还没有项目：

```bash
cd /srv/apps
git clone 你的仓库地址 zjuaaa-site
cd /srv/apps/zjuaaa-site
```

如果服务器上已经有项目：

```bash
cd /srv/apps/zjuaaa-site
git pull
```

---

## 3. 第一次迁移：创建环境变量

```bash
nano /srv/data/zjuaaa-site/.env
```

填入：

```env
DATABASE_URL="file:/srv/data/zjuaaa-site/dev.db"
AUTH_SECRET="你的随机字符串"
AUTH_URL="https://你的域名"
ADMIN_USERNAME="admin"
ADMIN_PASSWORD="你的后台密码"
ADMIN_DISPLAY_NAME="ZJUAAA Admin"
NODE_ENV="production"
UPLOAD_DIR="/srv/data/zjuaaa-site/uploads"
```

如果要生成随机字符串：

```bash
openssl rand -base64 32
```

---

## 4. 第一次迁移：链接 .env 到代码目录

```bash
rm -f /srv/apps/zjuaaa-site/.env
ln -s /srv/data/zjuaaa-site/.env /srv/apps/zjuaaa-site/.env
```

---

## 5. 第一次迁移：迁移旧数据库

如果旧数据库还在代码目录里：

```bash
cp /srv/apps/zjuaaa-site/prisma/dev.db /srv/data/zjuaaa-site/dev.db
cp /srv/apps/zjuaaa-site/prisma/dev.db-shm /srv/data/zjuaaa-site/ 2>/dev/null || true
cp /srv/apps/zjuaaa-site/prisma/dev.db-wal /srv/data/zjuaaa-site/ 2>/dev/null || true
```

如果你本来就还没有正式数据，可以跳过这一步。

---

## 6. 第一次迁移：迁移旧图片

如果旧图片还在：

```bash
mkdir -p /srv/data/zjuaaa-site/uploads
cp -r /srv/apps/zjuaaa-site/public/uploads/* /srv/data/zjuaaa-site/uploads/ 2>/dev/null || true
```

---

## 7. 第一次迁移：安装依赖并初始化

```bash
cd /srv/apps/zjuaaa-site
npm install
npx prisma db push --skip-generate
npx prisma generate
npm run db:seed-admin
```

---

## 8. 第一次迁移：构建并启动

```bash
cd /srv/apps/zjuaaa-site
npm run build
pm2 start npm --name zjuaaa-site -- start
pm2 save
```

如果已经存在旧进程，改用：

```bash
pm2 restart zjuaaa-site
```

查看状态：

```bash
pm2 status
pm2 logs zjuaaa-site --lines 30
```

---

## 9. Nginx 启用

创建配置：

```bash
nano /etc/nginx/sites-available/zjuaaa-site
```

写入：

```nginx
server {
    listen 80;
    server_name 你的域名 www.你的域名;

    client_max_body_size 25M;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用：

```bash
ln -s /etc/nginx/sites-available/zjuaaa-site /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

---

## 10. HTTPS

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名 -d www.你的域名
pm2 restart zjuaaa-site
```

## 11. 以后日常更新代码

```bash
cd /srv/apps/zjuaaa-site
git pull
npm install
npx prisma db push --skip-generate
npx prisma generate
npm run build
pm2 restart zjuaaa-site
pm2 logs zjuaaa-site --lines 30
```

---

## 12. 只重启服务

```bash
pm2 restart zjuaaa-site
```

---

## 13. 查看运行日志

```bash
pm2 logs zjuaaa-site
```

或者只看最后几十行：

```bash
pm2 logs zjuaaa-site --lines 50
```

---

## 14. 备份数据库

```bash
cp /srv/data/zjuaaa-site/dev.db /srv/backups/zjuaaa-site/dev-$(date +%F-%H%M).db
```

---

## 15. 备份上传图片

```bash
rsync -a /srv/data/zjuaaa-site/uploads/ /srv/backups/zjuaaa-site/uploads-$(date +%F-%H%M)/
```

---

## 16. 恢复数据库

假设你要恢复某个备份：

```bash
cp /srv/backups/zjuaaa-site/dev-2026-03-26-1200.db /srv/data/zjuaaa-site/dev.db
pm2 restart zjuaaa-site
```

恢复前建议先停服务：

```bash
pm2 stop zjuaaa-site
cp /srv/backups/zjuaaa-site/dev-2026-03-26-1200.db /srv/data/zjuaaa-site/dev.db
pm2 start zjuaaa-site
```

---

## 17. 恢复上传图片

```bash
rsync -a /srv/backups/zjuaaa-site/uploads-2026-03-26-1200/ /srv/data/zjuaaa-site/uploads/
pm2 restart zjuaaa-site
```

---

## 18. 检查端口和 Nginx

```bash
ss -tulpn | grep 3000
systemctl status nginx
nginx -t
```

---

## 19. 安全组 / 防火墙

如果你还没放行：

```bash
ufw allow OpenSSH
ufw allow 80
ufw allow 443
ufw enable
```

腾讯云控制台里也要放行：

- `22`
- `80`
- `443`
